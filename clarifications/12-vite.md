# Precisions sur l'architecture technique

Lors de la construction du projet, il faut privilegier les commandes d'installation ou d'initialisation.

Exemple:

```
bun create vite@latest
bun install express
```

## Docker

Utiliser docker et docker compose

## Bun

Utiliser Bun **comme gestionnaire de dépendances, exécuteur de scripts, et runtime**.

### Version

- **Version verrouillée : Bun 1.3.5** (dev, CI et prod).

## Vite

Dans le front-end, il faut utilliser la librairie Vite.

## Production

- Le serveur en production doit tourner sur **Bun (runtime)** (pas Node).
